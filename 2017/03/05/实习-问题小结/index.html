面向对象：

面向对象三个特性：

封装、继承、多态

补充:
python是动态语言，不要求严格的继承体系—鸭子类型。
子类可以向上看作父类类型，通过父类调用子类的同名方法实现多态。
多态：同一操作作用于不同的对象，可以有不同的解释，产生不同的执行结果。

类的三种方法

  • 实例方法：有self，表示创建的实例(对象)本身。通过‘实例名.方法名’a.foo(x)即foo(a,x)
  • 静态方法：在类成员函数前面加上@staticmethod，除了传入的参数，不要其他参数
  • 类方法：有cls，在类成员函数前面加上@classmethod，通过’类名.方法名‘调用，A.foo(x)即foo(A,x)

新式类和旧式类

为了统一类(class)和类型(type)。python2.2版本引进新式类，之前的类叫做经典类或旧式类。
新式类增加__slots__ 和 __getattribute__,多继承采用广度优先搜索BFS
Python 2.x中默认都是经典类。
Python 3.x中默认都是新式类。

为了确保自己使用的是新式类，有以下方法：

  • 把这个赋值语句放在类模块代码的最前面 __metaclass__ = type
  • 从内建类object直接或者间接地继承。

__init__ 和 __new__

创建一个新实例时调用__new__,初始化一个实例时用__init__,这是它们最本质的区别。
new方法会返回所构造的对象，init则不会.
new函数必须以cls作为第一个参数，而init则以self作为其第一个参数.

实例属性和类属性：

  • 实例属性：给实例绑定属性的方法是通过实例变量，或者通过self变量(在init)。
  • 类属性：在class中定义属性，这种属性是类属性，归Student类所有。
    注意：相同名称的实例属性将屏蔽掉类属性

super()

避免在子类中直接使用父类的名字

__slots__

在面向对象中，用__slots__来限制类的实例能够添加的属性，子类继承父类不会继承这个限制

元类metaclass

metaclass，元类，通俗点就是类的类，可以根据元类来创建类，创建元类要从type派生，在元类中定义函数def __new__(cls,name,bases,attrs)。应用在ORM框架，动态定义类。

函数问题

匿名函数lambda

关键字lambda表示匿名函数，冒号前面表示函数参数，匿名函数的限制就是只有一个表达式，返回值就是表达式的结果。
lambda有自己的命名空间，访问参数范围受限。

作用：简洁，避免函数名冲突、在需要传入函数的时候，不需要显式的定义函数，直接传入匿名函数。

属性访问限制

把属性的名称前加上两个下划线_，在Python中，实例的变量名如果以\_开头，就变成了一个私有变量。
以一个下划线开头的实例变量名，这样的实例变量外部是可以访问的，但是，按照约定俗成的规定，当你看到这样的变量时，意思就是，“虽然我可以被访问，但是，请把我视为私有变量，不要随意访问”。

  • __foo__:一种约定,Python内部的名字,用来区别其他用户自定义的命名,以防冲突.
  • _foo:一种约定,用来指定变量私有.程序员用来指定私有变量的一种方式.
  • __foo:这个有真正的意义:解析器用_classname__foo来代替这个名字,以区别和其他类相同的命名.

*args，**kwargs

  • 如果我们不确定要往函数中传入多少个参数，或者我们想往函数中以列表和元组的形式传参数时，那就使要用*args；
  • 如果我们不知道要往函数中传入多少个关键词参数，或者想传入字典的值作为关键词参数时，那就要使用**kwargs。

    5.python导入模块的搜索路径

    python会优先搜索主目录，当导入的文件跨目录时设置PAYTHONPATH很重要

其他

if __name__==’__main__‘:

在命令行中，python解释器会把__name__变量置为__main__，如果模块中有if __name__==’__main__’:这句话，那么他之后的操作会自动执行。
而当我们在其他的py文件里导入这个模块时，__name__的值为模块的名称，if条件不满足，后面的就不会执行。 用途就是测试一个模块。

变量作用域

python的作用域一共有四种：

  • L （Local） 局部作用域
  • E （Enclosing） 闭包函数外的函数中
  • G （Global） 全局作用域
  • B （Built-in） 内建作用域
    以 L –> E –> G –>B 的规则查找。
    Python除了module／def/class/lambda 外，其他如: if/elif/else/ try/except for/while并不能改变其作用域。
