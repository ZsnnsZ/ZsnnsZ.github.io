问题小结

发表于 2017-03-05

面向对象：

面向对象三个特性：

封装、继承、多态

补充:
python是动态语言，不要求严格的继承体系—鸭子类型。
子类可以向上看作父类类型，通过父类调用子类的同名方法实现多态。
多态：同一操作作用于不同的对象，可以有不同的解释，产生不同的执行结果。

类的三种方法

  • 实例方法：有self，表示创建的实例(对象)本身。通过‘实例名.方法名’a.foo(x)即foo(a,x)
  • 静态方法：在类成员函数前面加上@staticmethod，除了传入的参数，不要其他参数
  • 类方法：有cls，在类成员函数前面加上@classmethod，通过’类名.方法名‘调用，A.foo(x)即foo(A,x)

新式类和旧式类

为了统一类(class)和类型(type)。python2.2版本引进新式类，之前的类叫做经典类或旧式类。
新式类增加__slots__ 和 __getattribute__,多继承采用广度优先搜索BFS
Python 2.x中默认都是经典类。
Python 3.x中默认都是新式类。

为了确保自己使用的是新式类，有以下方法：

  • 把这个赋值语句放在类模块代码的最前面 __metaclass__ = type
  • 从内建类object直接或者间接地继承。

__init__ 和 __new__

创建一个新实例时调用__new__,初始化一个实例时用__init__,这是它们最本质的区别。
new方法会返回所构造的对象，init则不会.
new函数必须以cls作为第一个参数，而init则以self作为其第一个参数.

[ML]逻辑斯谛回归和最大熵模型

发表于 2017-03-02

Logistic Regression：逻辑斯谛回归
Maximum entropy model:最大熵模型
LR回归和最大熵模型都是对数线性模型。

LR

sklearn中与逻辑回归有关的类主要是LogisticRegression， LogisticRegressionCV 和logistic_regression_path，其中LogisticRegression和LogisticRegressionCV的主要区别是LogisticRegressionCV使用了交叉验证来选择正则化系数C。而LogisticRegression需要自己每次指定一个正则化系数。logistic_regression_path类则比较特殊，它拟合数据后，不能直接来做预测，只能为拟合数据选择合适逻辑回归的系数和正则化系数。主要是用在模型选择的时候

  • 正则化选择参数：penalty

    l2,l1

  • 优化算法选择参数：solver

    liblinear,lbgfs,newton-cg,sag

  • 类型权重参数： class_weight

    可以定义class_weight={0:0.9, 1:0.1}，这样类型0的权重为90%，而类型1的权重为10%。
    如果class_weight选择balanced，那么类库会根据训练样本量来计算权重。某种类型样本量越多，则权重越低，样本量越少，则权重越高。

定义：(逻辑斯蒂分布)设X是连续随机变量，X服从逻辑斯蒂分布是指X具有sigmoid形状的分布函数。以(μ，1/2)为中心对称。

在Ng的ppt中取g(z) = 1/(1+exp(-z))

设h(x) = g(wx) = P(Y=1|X) = exp(wx)/(1+exp(wx))

由sigmoid图像可以很简单的看出wx与P(Y=1|X)的关系

数学推导：对数几率log(P(Y=1|X)/(1-P(Y=1|X))) = wx
说明输出Y=1的对数几率是由输入x的线性函数表示的模型。

朴素贝叶斯原理

朴素贝叶斯的思想基础是这样的：对于给出的待分类项，求解在此项出现的条件下各个类别出现的概率，哪个最大，就认为此待分类项属于哪个类别

公式：由P(A∩B) = P(A) P(B|A) = P(B) P(A|B)
得到P(A|B) = P(B|A) * P(A) / P(B)
