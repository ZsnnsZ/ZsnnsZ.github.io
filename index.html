PCILearning

chapter2

1.基于用户的协作型过滤(user-based collaborative filtering)

适用于规模较小的、变化非常频繁的数据集

  • 1.1数据集构建使用嵌套字典，表达不同人及其偏好
  • 1.2寻找相近用户
  • 1.2.1欧几里得距离：二维空间几何距离
    策略：返回值 1/（d+1）
  • 1.2.2皮尔逊相关度
    作用：判断两组数据与某一直线的拟合程度，修正‘夸大分值’的情况
    适用场合：数据不是很规范时（一组普遍偏高，一组普遍偏低）
  • 1.2.3曼哈顿距离（A*寻路算法）
  • 1.2.4Jaccard系数
  • 1.3推荐物品
    对相似用户对物品的评分进行加权平均
  • 1.4寻找相似物品
    矩阵转置

2.基于物品的协作型过滤(item-based collaborative filtering)

在拥有大量数据集的情况下，基于物品的协作型过滤更好。
物品间的比较不像用户间的比较那么频繁。

优点：大量计算任务预先执行，更快的给予需要推荐的用户结果
缺点：增加了维护物品相似度表达的额外开销

具体说明：在为某用户提供推荐时，先统计他曾评分的物品集，寻找这些物品的相似物品集，进行加权平均

Beautifulsoup使用

发表于 2017-02-19

创建BeautifulSoup对象

  • 导入bs4库

  • 创建beautifulsoup对象

  • 格式化打印soup对象内容

四大对象种类

  1. tag(两个重要属性是name和attrs)

  2. NavigableString

    • 获取标签内部的文字

  3. BeautifulSoup
    表示的是一个文档的全部内容.大部分时候,可以把它当作 Tag 对象，是一个特殊的 Tag

  4. Comment
    Comment 对象是一个特殊类型的 NavigableString 对象，其实输出的内容仍然不包括注释符号

遍历文档树

  1. .contents
    tag 的 .content 属性可以将tag的子节点以列表的方式输出

  2. .children
    它返回的不是一个 list，不过我们可以通过遍历获取所有子节点
