python基础：

1.面向对象三个特性：

封装、继承、多态

补充:
python是动态语言，不要求严格的继承体系—鸭子类型。
子类可以向上看作父类类型，通过父类调用子类的同名方法实现多态。
多态：同一操作作用于不同的对象，可以有不同的解释，产生不同的执行结果。

2.list遍历：

l = []
- for x in l:
      print x
- for i in range(len(l)):
      print l[i]
- for i,x in enumerate(l):
      print i,x

补充：
list是有序可重复集合，删除可用pop(i)、del、remove

3.lambda

关键字lambda表示匿名函数，冒号前面表示函数参数，匿名函数的限制就是只有一个表达式，返回值就是表达式的结果。
lambda有自己的命名空间，访问参数范围受限。

作用：简洁，避免函数名冲突、在需要传入函数的时候，不需要显式的定义函数，直接传入匿名函数。

什么时候用lambda？

4.类的三种方法

  • 实例方法：有self，表示创建的实例(对象)本身。通过‘实例名.方法名’a.foo(x)即foo(a,x)
  • 静态方法：在类成员函数前面加上@staticmethod，除了传入的参数，不要其他参数
  • 类方法：有cls，在类成员函数前面加上@classmethod，通过’类名.方法名‘调用，A.foo(x)即foo(A,x)

5.yield

理解yield之前要先理解iterables和generators

  • iterables：可以用在for…in…语句中的都是可迭代的:比如lists,strings,files…因为这些可迭代的对象你可以随意的读取,但是你必须把它们的值放到内存里,当它们有很多值时就会消耗太多的内存.
  • generators：生成器也是迭代器的一种,但是你只能迭代它们一次.原因很简单,因为它们不是全部存在内存里,它们只在要调用的时候在内存里生成。

直接定义生成器和迭代器的区别就是用()代替[],还有你不能用for i in mygenerator第二次调用生成器 

# 这里你创建node方法的对象将会返回一个生成器
def node._get_child_candidates(self, distance, min_dist, max_dist):

  # 这里的代码你每次使用生成器对象的时候将会调用

  if self._leftchild and distance - max_dist < self._median:
      yield self._leftchild

  if self._rightchild and distance + max_dist >= self._median:
      yield self._rightchild

  # 如果代码运行到这里,生成器就被认为变成了空的

# 创建空列表和一个当前对象索引的列表
result, candidates = list(), [self]

# 在candidates上进行循环(在开始只保含一个元素)
while candidates:

    # 获得最后一个condidate然后从列表里删除
    node = candidates.pop()

    # 获取obj和candidate的distance
    distance = node._get_dist(obj)

    # 如果distance何时将会填入result
    if distance <= max_dist and distance >= min_dist:
        result.extend(node._values)

    candidates.extend(node._get_child_candidates(distance, min_dist, max_dist))

return result

使用了yield的函数在yield的地方就会返回一个生成器对象，当你调用函数的时候,函数里的代码并没有运行.函数仅仅返回生成器对象,这就是它最微妙的地方。

python补充：

1.实例属性和类属性：

  • 实例属性：给实例绑定属性的方法是通过实例变量，或者通过self变量(在init)。
  • 类属性：在class中定义属性，这种属性是类属性，归Student类所有。
    注意：相同名称的实例属性将屏蔽掉类属性

2.list和dict、set

  • dict有以下几个特点：
    查找和插入的速度极快，不会随着key的增加而变慢；
    需要占用大量的内存，内存浪费多。

  • 而list相反：
    查找和插入的时间随着元素的增加而增加；
    占用空间小，浪费内存很少。

set是数学意义上的无序和无重复元素集合，交&， 并|

3.访问限制

把属性的名称前加上两个下划线，在Python中，实例的变量名如果以开头，就变成了一个私有变量。
以一个下划线开头的实例变量名，这样的实例变量外部是可以访问的，但是，按照约定俗成的规定，当你看到这样的变量时，意思就是，“虽然我可以被访问，但是，请把我视为私有变量，不要随意访问”。

  • __foo__:一种约定,Python内部的名字,用来区别其他用户自定义的命名,以防冲突.
  • _foo:一种约定,用来指定变量私有.程序员用来指定私有变量的一种方式.
  • __foo:这个有真正的意义:解析器用_classname__foo来代替这个名字,以区别和其他类相同的命名.

4.args，*kwargs

  • 如果我们不确定要往函数中传入多少个参数，或者我们想往函数中以列表和元组的形式传参数时，那就使要用*args；
  • 如果我们不知道要往函数中传入多少个关键词参数，或者想传入字典的值作为关键词参数时，那就要使用**kwargs。

5.python中的元类

metaclass，元类，通俗点就是类的类，可以根据元类来创建类，创建元类要从type派生，在元类中定义函数def new(cls,name,bases,attrs)。应用在ORM框架，动态定义类。

6.python中的装饰器

函数也是对象，可以将函数作为参数，在代码运行期间动态增加功能的方式，称之为“装饰器”（Decorator） 

# 字体变粗装饰器,（不传参）
def makebold(fn):
    # 装饰器将返回新的函数
    def wrapper():
        # 在之前或者之后插入新的代码
        return "<b>" + fn() + "</b>"
    return wrapper

# 日志打印器，(传参)
>>> def log(fn):
...     def wrapper(*args, **kw):
...             print 'call %s():' % fn.__name__
...             fn(*args, **kw)
...     return wrapper
>>> @log
... def now():
...     print 'x'
... 
>>> now()
call now():
x

7.python中的偏函数

这是模块functools里面的一个函数，functools.partial的作用就是，把一个函数的某些参数给固定住（也就是设置默认值），返回一个新的函数，调用这个新函数会更简单。

8.if __name__==’__main__‘:

在命令行中，python解释器会把__name__变量置为__main__，如果模块中有if __name__==’__main__’:这句话，那么他之后的操作会自动执行。
而当我们在其他的py文件里导入这个模块时，__name__的值为模块的名称，if条件不满足，后面的就不会执行。 用途就是测试一个模块。

朴素贝叶斯原理

朴素贝叶斯的思想基础是这样的：对于给出的待分类项，求解在此项出现的条件下各个类别出现的概率，哪个最大，就认为此待分类项属于哪个类别

公式：由P(A∩B) = P(A) P(B|A) = P(B) P(A|B)
得到P(A|B) = P(B|A) * P(A) / P(B)
